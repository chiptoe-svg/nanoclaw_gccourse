Build-Your-Own Photo-Critique Agent

An instructional design brief for an AI-assisted unit in which students construct, rather than merely operate, a photographic-critique agent. The instructor defines the constraints and the evaluative floor; each student authors the agent's identity, persona, and tool access, then applies it to their own photographs.

AudienceCourse instructor UnitAssignment 3 — Take Your Best Shot DurationOne week (two lab sessions) PlatformNanoClaw classroom · Agent Playground

Status: draft for instructor review. Figures and tool claims dated mid-2026; items marked for verification appear in §8.

§1
Pedagogical rationale

Instructor-defined constraints; student-authored agents

The unit's premise is constructionist: students learn the elements of critique by having to specify them. Authoring an agent requires converting tacit visual judgment into explicit criteria and questions. Control is divided into two clearly bounded roles.

Instructor defines
  • The evaluative floor — a Socratic critique stance, tone expectations, photojournalism ethics, and the required submission format. Authored once and propagated to every student.
  • The available tools — which skills students are permitted to enable (for example, web search).
  • Assessment — the rubric and a reference exemplar defining a strong build.
  • Administration — roster, per-student accounts, model and cost settings.
Each student authors
  • Identity — the agent's name, and a brief statement of its character and voice.
  • Persona — its critique stance, the visual elements it prioritizes, and a bank of Socratic questions written in the student's own words.
  • Tool access — enabling or disabling the available skills and observing the effect on the agent's behavior.
  • Evaluation in use — running the agent on their own photographs and revising it until its critique is useful.
Why construction rather than consumption. Specifying what an agent should ask forces a student to define what good critique consists of. The exercise externalizes the student's developing critical vocabulary; the agent functions as an instrument for examining their own judgment, not as an authority that supplies answers.
§2
Persona architecture

A three-layer persona model

The platform composes personas in the order global → role → per-agent. The instructor floor is inherited by every student agent, so a weak or incomplete student persona still operates within the required pedagogical and ethical bounds.

Instructor floor
Shared and locked. Socratic-only critique, tone and conduct expectations, the photojournalism ethics rule, and the required submission format. A single file, propagated to all students.
Student persona
The student's work. Identity, weighted visual criteria, and a Socratic question bank — entered in the Persona tab from a provided scaffold and worksheet.
Assignment handout
The wrapper. The student-facing document guiding the full sequence: build, photograph, critique, reshoot.

§4
The build

A seven-part persona scaffold

Authoring the agent is a writing task, completed in the Persona tab and prompted by a reflective worksheet. The seven components:

i

Identity

Name, character, and voice.

ii

Critique stance

Degree of directness; how far to question before offering a hint.

iii

Opening move

Greeting and the required technical-or-creative branch.

iv

Weighted criteria

Selecting and ranking composition, focus and depth of field, light, framing, moment, and background.

v

Question bank

The central task: two to three Socratic questions per criterion, posed rather than answered.

vi

Reshoot guidance

Helping the student select images to reshoot; producing the feedback summary.

vii

Handoff

Producing the submission summary the student retains.

§5
Course sequence

Two lab sessions

Lab A — Construct and plan
  1. Draw a campus building at random.
  2. Introduction to the Playground; construct and test the critique agent on practice images.
  3. Enable and disable a skill; compare agent and unaided-model behavior.
  4. Prepare the fifty-image shot list.
Lab B — Critique and reshoot plan
  1. Import and cull to ten images in Lightroom Classic.
  2. Apply the student's own agent to the ten; work through the Socratic critique.
  3. Produce the feedback summary and submission record.
  4. Plan the four reshoots.

Between sessions: photograph the fifty images · After Lab B: reshoot the selected four

§6
Assignment 5 — studio reshoot

Image generation for previsualization: three approaches

In the studio reshoot (Assignment 5), generated images serve as a previsualization aid — a way to study lighting, posing, framing, and mood before a live shoot — not as a final deliverable. For this purpose the relevant criteria are institutional access, the precision of lighting control, photorealism, the provenance of training data, and instructional value. The original brief specified Adobe Firefly; two further approaches merit consideration. The instructor's recent note that the university holds an OpenAI license with student credits is material to the comparison.

ChatGPT Images 2.0 (GPT Image 2) — OpenAI

Proposed primary tool

Reported (April 2026) as a reasoning-based generator on the GPT-5.4 backbone, producing up to 4K output and accepting reference images, with photorealism rated at the top of independent comparison leaderboards. Because it can reason over a brief and a reference image, it pairs directly with the companion lighting agent (§7): the agent can produce a setup description and a ready-to-use generation prompt. Access on campus is via the institution's existing OpenAI license and student credits, removing the cost and account friction of consumer alternatives.

Considerations. The model post-dates this author's training data; capability claims are drawn from the sources in §9 and should be confirmed (§8). Training-data provenance and commercial-use terms are less transparent than Firefly's. Lighting control is prompt- and reference-driven rather than physically parameterized, so it is well suited to mood and overall look but less precise for isolating a single lighting variable.

Adobe Firefly — Adobe

Alternative — integrated and commercially safe

The original choice, and defensible on grounds independent of raw photorealism. Firefly is trained on Adobe Stock and licensed or public-domain material, which Adobe positions as commercially safe — a meaningful consideration for coursework. It is available to students through their university accounts and is integrated into Photoshop (generative fill and expand), a tool already taught in the course, so it introduces no new application. Localized edits give moderate control over discrete changes such as adding a rim light or altering a background.

Considerations. Photorealism is generally rated below the current OpenAI model and below dedicated photorealistic models. Lighting control is prompt- and edit-based, not true relighting.

IC-Light + ControlNet on Stable Diffusion

Advanced — instructor-led lighting demonstration

Makers. Stable Diffusion is an open-weight model from Stability AI. ControlNet (Zhang, Rao & Agrawala, ICCV 2023) is an open-source method that conditions the model on structural inputs — human pose, depth, surface normals, edges — so subject and scene geometry can be fixed while other variables change. IC-Light ("Imposing Consistent Light," Lvmin Zhang, 2024) is an open-source relighting model from the same author.

Why it teaches lighting control. Unlike prompt-driven generators, IC-Light lets a student hold the subject constant and vary only the illumination — specifying light direction, quality, color temperature, and an implied source or background, and producing a consistent relit result. This makes light an isolated, manipulable variable, which maps directly onto the studio-lighting curriculum: a single portrait can be re-rendered as key-left versus key-right, with and without fill, with and without a rim or hair light, at high versus low ratio, and under warm versus cool sources. Students can compare canonical patterns — split, loop, Rembrandt, butterfly, broad and short lighting — and three-point setups side by side, on the same face, observing cause and effect in a way a live studio rarely permits within one session. ControlNet adds the complementary control of fixing pose and framing so that only lighting differs between comparisons.

Feasibility on available hardware. The instructor has access to two candidate diffusion stations separate from the classroom host: an NVIDIA DGX Spark (GB10, 128 GB) and a Mac Studio (M4 Max, 64 GB). IC-Light v1 is based on the lightweight Stable Diffusion 1.5, well within either machine's capacity. The two trade off differently. The DGX Spark is CUDA-native, so the open-source toolchain (ComfyUI, the ControlNet preprocessors, the IC-Light nodes, and their optimizations) installs and runs without the occasional gaps encountered under Apple's Metal backend; its 128 GB also allows substantially larger models (e.g. Flux) and heavy batching. Its memory bandwidth (273 GB/s), however, is the limiting factor for single-image latency, which is bandwidth-bound; the M4 Max, at roughly double that bandwidth, may render an individual SD 1.5 frame as quickly or quicker, at the cost of the Apple-Silicon toolchain caveats. With a dedicated station the principal constraints shift from compute to (1) toolchain setup and maintenance and (2) interactive throughput for a whole class at once. The practical role is therefore a shared or small-group generation station — by rotation, queue, or batch — rather than an instructor-only demonstration or a per-student tool. A no-install browser demonstration remains available as a fallback (§9).

Recommended configuration. Adopt ChatGPT Images 2.0 as the primary previsualization tool, given institutional access and current image quality; retain Adobe Firefly as the integrated, commercially-safe alternative; and operate IC-Light with ControlNet on a dedicated diffusion station (the DGX Spark being the more compatible host) as a shared resource for controlled-relighting exercises. The three are complementary rather than competing, and the recommendation does not depend on any single vendor remaining ahead on image quality.

§7
Companion agent

Studio lighting reverse-engineer (Assignment 5)

A second agent, following the same construction pattern, takes an inspiration photograph and reverse-engineers a plausible production setup: a lighting diagram (key, fill, rim), modifiers and ratios, camera settings, lens, framing, and posing — a plan for the real studio shoot. Its text output complements the image-generation step: the agent produces the setup description and a generation prompt; the generator (§6) produces the visual reference. This module will be specified in full after Assignment 3 is complete.

§8
Computing environment & open questions

Environment, and items to verify

Host

Mac Studio (M1 Ultra, 32 GB). Orchestrates a short-lived container per chat message.

Models

Cloud-side (Claude via the Anthropic API) for the agents; the host does not run the language model.

Local diffusion

Optional dedicated station for §6: DGX Spark (128 GB, CUDA-native) or M4 Max (64 GB). Shared/small-group use, not a per-student service. No general-purpose local language model.

Concurrency

Monitor container spin-up when a full class uses the agent simultaneously; stagger if needed.

Portability

All materials are plain markdown; the exercise degrades to a self-built Claude Project if the platform is not deployed.

Export

Low-technology: the agent prints a submission summary in chat, which the student retains. No export skill is built at this stage.

  1. JPEG export settings. The handout references unspecified export settings (color space, long-edge dimension, quality); exact values are required.
  2. Skill toggling. Confirm the student lockdown permits enabling and disabling instructor-installed skills.
  3. Skill selection. Determine which skill(s) to pre-install for the comparison exercise (web search assumed).
  4. Image-model claims. Confirm current ChatGPT Images 2.0 capabilities, access terms, and standing, which post-date the author's training data.
  5. Deployment. NanoClaw classroom or the standalone Claude-Project fallback; affects only the setup guide, not the core materials.
  6. Lab timing. Confirm the two-session sequence matches actual lab scheduling.
