🦞 Install NanoClaw on Mac Studio

Three commands, click-to-copy. Tested for macOS 13+ with Claude Code already authenticated.

Before you start

You need:

  • have it Claude Code installed and authenticated (you confirmed)
  • have it GitHub authenticated via gh auth login or SSH (you confirmed)
  • check Docker Desktop for Mac — install if missing: docker.com/products/docker-desktop. Apple Silicon: pick the Apple Silicon build.

Apple Container alternative: if you'd rather use Apple's native container runtime instead of Docker (M-series only, faster cold-start, smaller install), skip Docker. After install, run /convert-to-apple-container in Claude Code.

1

Clone the repo

This pulls the v3 playground branch directly. Once merged to main, you can drop the --branch flag.

git clone --branch worktree-playground-v3 https://github.com/chiptoe-svg/nanoclaw_gccourse.git ~/nanoclaw cd ~/nanoclaw
2

Run the bootstrap script

nanoclaw.sh installs Node + pnpm + native modules, builds the agent container, walks you through credential and channel setup, and pairs your first messaging app. If a step fails it hands off to Claude Code to diagnose and resume from where it broke.

bash nanoclaw.sh

Expect ~5–15 minutes of mostly-automated work. You'll be prompted for: your display name, AI-coding-CLI (pick Claude Code), provider credentials, timezone, and which channel to pair first (Telegram is the smoothest).

3

Open the playground

After step 2 finishes, the host service is running as a LaunchAgent and the playground is reachable at:

http://<your-mac-studio-LAN-ip>:3002/

The first time you visit, run /playground in your paired chat (Telegram, etc.) — you'll get a magic-link URL DM'd back. Click it, you'll land on the Home tab of the new v3 surface.

What goes in .env (for reference)

nanoclaw.sh writes most of this for you. Listed here so you know what's controlling what — useful when troubleshooting or layering on options later.

Required (one of)

VariablePurpose
ANTHROPIC_API_KEYAnthropic API key — the simplest path. Get from console.anthropic.com.
CLAUDE_CODE_OAUTH_TOKENUse your Claude Code subscription instead of an API key. Setup script can extract this from your local ~/.claude/.credentials.json.

Optional providers

VariablePurpose
OPENAI_API_KEYFor agents using Codex (OpenAI) as their provider. Add via /add-codex in Claude Code.
ANTHROPIC_BASE_URL / OPENAI_BASE_URLCustom upstream URLs (proxies, local models). Defaults to the official endpoints.

Playground

VariableDefaultPurpose
PLAYGROUND_ENABLED(unset)Set to 1 to allow the /playground Telegram command. Required.
PLAYGROUND_PORT3002HTTP port for the playground web UI.
PLAYGROUND_BIND_HOST0.0.0.0Set to 127.0.0.1 if you'd rather tunnel via SSH and skip magic-link auth.
PLAYGROUND_IDLE_MINUTES30Auto-revoke idle playground sessions after N minutes.

General

VariableDefaultPurpose
ASSISTANT_NAMEAndyWhat your agents call themselves and how you address them (@Andy).
TZauto-detectedTimezone for scheduled tasks and message formatting (IANA, e.g. America/New_York).
CONTAINER_TIMEOUT1800000Max container lifetime in ms (default 30 min).
MAX_CONCURRENT_CONTAINERS5Cap on simultaneous agent containers.

Once installed

From the Mac Studio terminal, useful follow-ups (all via Claude Code in the repo dir):

CommandWhat it does
claude → /setupRe-runs setup interactively (idempotent — safe to re-invoke if something needs redoing).
claude → /add-telegramPair a Telegram bot if you skipped channels in step 2.
claude → /init-first-agentBootstrap the first agent group for the channel you just paired.
claude → /debugIf anything breaks. Looks at logs and runs container diagnostics.
tail -F logs/nanoclaw.logWatch the host service.
launchctl kickstart -k gui/$(id -u)/com.nanoclawRestart the service after a .env change.

Troubleshooting

SymptomFix
Step 2 fails at "Build container"Open Docker Desktop, accept any license prompts, retry: bash nanoclaw.sh (it resumes from the failing step).
Magic-link 404sMake sure PLAYGROUND_ENABLED=1 in .env and restart: launchctl kickstart -k gui/$(id -u)/com.nanoclaw.
"current agent: (no agent yet)" in Home tabRun /init-first-agent in Claude Code from the repo dir.
Magic-link URL points at localhost instead of the LAN IPSet PLAYGROUND_PUBLIC_HOST=<mac-studio-lan-ip> in .env and restart.
Anything elseInside the repo dir: claude → /debug. It reads logs and offers fixes.
